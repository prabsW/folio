Process

Collecting Data

Find out about the three Ps (people, problem, project)

Create user stories and personas/profile backed up with data. Design best user flow with studies, benchmarking and trend analysis. Every project is different and its for different group of people. Once we have data of our users then we can fuilfil there goals.

Get ready to design

Sketching on paper or with tool like axure, uxpin etc

Test lo-fi prototypes. Gather material informations. Then find a design worth refining. Do multiple templates generic sketechs.

Approval / fighting over picky details

This part is for "buy-in" process.

Noone would want to see works go directly to the trash. Approval reduce risk of starting over and it also let stakeholders know the story behind design. This could be done as presentation, signoff etc.

Start building or designing

This part is for interaction design

After sign-off initial design build high-fi prototypes and test.
