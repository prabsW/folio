Sparkle Cupcakery

Improve clients website user experience. Usually UX process starts with either feedback or findings. In this case bad feedback, almost 50% costumers gave bad usability feedback. I was assigned to improve Sparkle’s user experience. Soon after analyzing project I found serious issues with navigation, info page, order forms were all over, delivery page wasn’t working and one can't pick a date of order.

Things I’ve done to improve UX.
1. Removed all other colors and only kept black and white
2. Simplify navigations added top nav, main nav and shopping cart nav
3. Add cake message, qty, amount, and greeting card message on more info page
4. Added pick up date, delivery date and configured wanted new
After doing this costumers complains were reduced by 95%

Process

Collecting Data

Find out about the three Ps (people, problem, project)

Create user stories and personas backed up with data. Design best user flow. Figure out users goal and the easiest way to get them there. Every project is different one approach would not work for all but if we start analysing data we'd get better user experience sooner. Everything has user experience our job is to make it better.

Get ready to design

Sketching on paper or with tool like axure, uxpin etc

Test lo-fi prototypes. Gather material informations. Then find a design worth refining. Do multiple templates generic sketechs.

Approval / fighting over picky details

This part is for "buy-in" process.

Noone would want to see works go directly to the trash. Approval reduce risk of starting over and it also let stakeholders know the story behind design which healps them invested/buy-in to the project. This could be done as presentation, signoff etc.

Start building prototype

Test Test Test

After sign-off initial design build high-fi prototypes and test. For interaction designs you want to nail all the interaction before hand it on to developers.
